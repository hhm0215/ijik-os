# 세션 마무리

이 프로젝트의 기억은 파일이다. 세션이 끝날 때 상태를 파일에 남겨야 다음 세션이 이어진다.

## 절차 (순서대로)

1. **변경과 품질 게이트 확인**:
   ```bash
   git status --short --branch
   npm run lint
   npm run build
   git diff --check
   ```
   사용자 흐름 변경은 `/run-app`, 파이프라인 변경은 `/pipeline-eval`까지 실행한다. 실패하면 자동 푸시하지 않고 고치거나 `PROGRESS.md`에 미완 상태와 재개 지점을 남긴다.

2. **IDEAS.md 정리**:
   - 이번 세션에서 반영한 항목 → `[done]`으로 변경
   - 안 하기로 결정한 항목 → `[dropped]` + 이유 한 줄
   - 세션 중 나온 새 아이디어/발견 → `[open]`으로 추가

3. **docs/PROGRESS.md 갱신**:
   - "현재 상태" 표를 최신으로 (단계, 다음 마일스톤, 차단 요소)
   - "세션 로그"에 새 블록 추가: 날짜 — 제목 / 한 것 / 확인된 것 / 발견된 이슈 / 다음
   - 단계가 완료됐으면 docs/PLAN.md §6 로드맵 표의 상태도 갱신

4. **커밋**:
   ```bash
   git add -A && git status --short   # 의도치 않은 파일이 없는지 확인
   git commit -m "<타입>: <요약>"
   git push origin HEAD
   ```
   - 검증된 작업 단위는 문서 갱신과 커밋 후 origin에 푸시한다. 권한·네트워크 실패 시 상태와 재개 지점을 보고한다.

5. **보고**: 커밋 해시와 푸시 여부, 반영된 IDEAS 항목 수, 다음 세션이 시작해야 할 지점을 간결하게 요약.

## 주의

- PROGRESS.md 세션 로그는 "다음 세션의 나"가 읽는 글이다 — 코드명/약어 대신 완전한 문장으로
- 데이터베이스(`data/`)는 gitignore 상태 — 커밋에 포함되지 않는 것이 정상
