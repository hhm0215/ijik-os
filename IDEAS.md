  이력서/자소서/포트폴리오와 직접 붙여넣은 내용을 경험 카드 후보로 분리한다. 원문 근거와

  원문 인용이 실제 입력에 포함됐는지 표시한다. 사용자가 고른 카드는 단일 DB 트랜잭션으로
  일괄 저장해 중간 실패·재시도 시 일부 카드만 남거나 중복되는 위험을 줄였다. 인용 검증과
  일괄 입력 경계는 TypeScript 단위 테스트로 고정했다.
- [done] (2026-07-15) **같은 경험이 업무·성과별로 과도하게 분할됨**: 프로젝트·역할·기간을
  가져오기 전용 구조화 식별자로 분리하고, 셋이 같은 후보는 서버 후처리에서 한 장으로
  통합한다. 여러 업무·성과는 분리 근거가 아니며 프로젝트·역할·기간 경계가 명확할 때만
  나누도록 프롬프트를 고쳤다. 화면에는 선택 후보 병합을 추가했고 단위 테스트와 저장 없는
  실제 Ollama 회귀 평가 명령으로 정책을 고정했다.
- [done] (2026-07-14) **화면 구성·디자인 UX 개편**: 사용자가 직접 "눈이 아프고 UX가

  카드 작성·분석 상세를 하나의 밝은 워크스페이스 디자인으로 통일하고, 정보 위계·여백·

- [open] (2026-07-11) **Windows Ollama 0.13.1에서 구조화 출력 실패**: 대형 스키마(analysisSchema) 호출 시 `failed to load model vocabulary required for format` + 서버 로그 `error parsing grammar: unexpected end of input`. 작은 스키마(추출)는 성공. Mac(0.31.1)에서는 재현 안 됨 — Windows Ollama 버전 업그레이드로 해결 시도가 1순위
- [done] (2026-07-12) **재분석 시 이전 결과 미정리 버그 수정**: persist에서 기존 결과 삭제(requirements/askbacks/drafts/interview, 자식은 FK cascade) + 신규 저장을 한 트랜잭션으로. requirements 사전 INSERT도 트랜잭션 안으로 이동.
